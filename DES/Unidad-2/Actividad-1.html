Actividad 2.1: Anatomía de una Petición HTTP con Telnet

Esta actividad práctica nos permitirá "hablar" directamente con un servidor web utilizando el protocolo HTTP en su forma más pura. Usaremos el cliente Telnet para enviar una petición manual y analizar la respuesta del servidor, desmitificando el proceso que ocurre cada vez que navegamos por internet.

Preparación: Habilitando el Cliente Telnet

En la mayoría de los sistemas operativos modernos, el cliente Telnet viene deshabilitado por defecto. A continuación se indica cómo activarlo:

  • Windows:
    1. Abra el "Panel de control".
    2. Diríjase a "Programas" y luego a "Activar o desactivar las características de Windows".
    3. En la lista que aparece, marque la casilla "Cliente Telnet" y haga clic en "Aceptar".
  • Debian, macOS y Linux:

    Por lo general, Telnet está disponible a través de la terminal. Si no es así, se puede instalar utilizando el gestor de paquetes de la distribución (por ejemplo, sudo apt-get install telnet en sistemas basados en Debian/Ubuntu).

Actividad: Realizando una Petición HTTP a un Servidor Web

Una vez que el cliente Telnet esté habilitado, puedes iniciar la actividad desde la línea de comandos (Símbolo del sistema en Windows, o Terminal en Debian, macOS y Linux).

Paso 1: Conexión al Servidor Web

El primer paso es establecer una conexión con un servidor web en el puerto estándar para HTTP, que es el puerto 80. Para esto, utilizaremos un sitio web example.com.

Abra su terminal y escriba el siguiente comando, seguido de la tecla Enter:

telnet example.com 80

Si la conexión es exitosa, la pantalla se quedará en blanco o negro, esperando sus instrucciones.

Paso 2: Enviando una Petición HTTP

Ahora es el momento de escribir una petición HTTP. Comencemos con la petición más común, un GET. Escribe las siguientes líneas exactamente como se muestran y presiona Enter después de cada una. Es importante presionar Enter dos veces al final para indicarle al servidor que ha terminado de enviar la petición. 

GET / HTTP/1.1
Host: example.com

Análisis de la Petición:

  • GET / HTTP/1.1: Esta es la línea de solicitud.
    • GET: Es el método HTTP que indica que queremos obtener un recurso.
    • /: Es la ruta del recurso que estamos solicitando. En este caso, es la página principal del sitio.
    • HTTP/1.1: Es la versión del protocolo HTTP que estamos utilizando.
  • Host: example.com: Esta es una cabecera obligatoria en HTTP/1.1 que especifica el dominio del servidor al que nos estamos conectando.

Paso 3: Observando la Respuesta del Servidor

Inmediatamente después de presionar Enter dos veces, el servidor web responderá con un mensaje HTTP. La respuesta tendrá una estructura similar a la siguiente:

HTTP/1.1 200 OK
Content-Type: text/html; charset=UTF-8
Content-Length: 1256
Date: Tue, 21 Oct 2025 14:30:00 GMT
Server: ECS (dcb/7F83)
Last-Modified: Tue, 17 Oct 2023 07:18:26 GMT
ETag: "3147526947"
Accept-Ranges: bytes
Connection: close

<!doctype html>
<html>
<head>
    <title>Example Domain</title>
    ... (resto del código HTML de la página)
</html>

Análisis de la Respuesta:

La respuesta HTTP se divide en tres partes principales:

  • Línea de Estado: HTTP/1.1 200 OK.
    • HTTP/1.1: La versión del protocolo.
    • 200: El código de estado, que en este caso indica que la petición fue exitosa ("OK").
    • OK: Un mensaje descriptivo del código de estado.
  • Cabeceras de Respuesta: Estas líneas proporcionan metadatos sobre la respuesta del servidor.
    • Content-Type: Indica el tipo de contenido que se está enviando (en este caso, un documento HTML).
    • Content-Length: El tamaño del cuerpo de la respuesta en bytes.
    • Date: La fecha y hora en que se generó la respuesta.
    • Server: Información sobre el software del servidor web.
  • Cuerpo de la Respuesta: Después de una línea en blanco, se encuentra el cuerpo del mensaje, que en el caso de una petición GET exitosa a una página web, será el código HTML de la misma.

Explorando Otros Métodos HTTP

Además de GET, puedes experimentar con otros métodos HTTP para observar diferentes comportamientos:

  • HEAD: Solicita solo las cabeceras de la respuesta, sin el cuerpo del mensaje. Esto es útil para verificar si un recurso existe sin necesidad de descargarlo por completo. La sintaxis es similar a GET: 
    HEAD / HTTP/1.1
Host: example.com
  • POST: Se utiliza para enviar datos a un servidor para crear un nuevo recurso. Su estructura es más compleja ya que incluye un cuerpo de mensaje con los datos a enviar.
  • PUT: Se emplea para actualizar un recurso existente en el servidor.
  • DELETE: Sirve para eliminar un recurso específico.

Realizar esta actividad con Telnet proporcionará una comprensión práctica y fundamental de la comunicación cliente-servidor en la web, desmitificando el proceso que ocurre cada vez que navega por internet.
