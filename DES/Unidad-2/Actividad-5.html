Estos desafíos requieren que investigues y apliques soluciones a problemas más complejos y realistas.

Desafío 7: Asegurando el Servidor con HTTPS y Let's Encrypt

Para este desafío, necesitarás un dominio real o un subdominio que apunte a la IP pública de tu servidor (puedes usar servicios de DNS dinámico si trabajas desde casa).

  1. Instala Certbot, el cliente oficial de Let's Encrypt, en tu servidor.
  2. Utiliza Certbot para obtener e instalar automáticamente un certificado SSL/TLS para uno de tus dominios.
  3. Certbot modificará tu configuración de Nginx. Examina los cambios que ha realizado en el archivo de Server Block.
  4. Verifica que tu sitio ahora carga con HTTPS y que el navegador muestra el candado de seguridad.
  5. Como paso final, asegúrate de que todo el tráfico que llega por HTTP (puerto 80) sea redirigido automáticamente a HTTPS (puerto 443). Certbot suele ofrecer esta opción durante la instalación.

Desafío 8: Nginx como Proxy Inverso para una Aplicación Node.js

  1. Instala Node.js en tu servidor.
  2. Crea una aplicación "Hello World" muy simple con Express.js que se ejecute en el puerto 3000.
  3. Modifica la configuración de Nginx para que actúe como proxy inverso. La idea es que cuando un usuario visite tu dominio en el puerto 80, Nginx intercepte la petición y la reenvíe a la aplicación Node.js que está escuchando en localhost:3000.
  4. Asegúrate de que puedes ver el mensaje de tu aplicación Node.js al acceder a tu dominio desde el navegador, sin necesidad de especificar el puerto :3000.

6. Tareas Adicionales: Asegurando el Servidor

6.1. Habilitar HTTPS con Certificados Autofirmados

Como práctica, genera un certificado SSL autofirmado y modifica el Server Block de tu sitio para que escuche en el puerto 443 (HTTPS). Esto te permitirá acceder a tu web de forma cifrada. Investiga los comandos necesarios para generar el certificado y las directivas de Nginx para usarlo (listen 443 ssl;, ssl_certificate;, ssl_certificate_key;).

6.2. Redirección de HTTP a HTTPS

Una vez que HTTPS funcione, el sitio será accesible tanto por HTTP como por HTTPS. La configuración correcta es forzar a todos los usuarios a usar la conexión segura. Modifica tu Server Block para que cualquier petición que llegue por el puerto 80 (HTTP) sea redirigida automáticamente al puerto 443 (HTTPS).
