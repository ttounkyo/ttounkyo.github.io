Actividad 2.4. Transferencia de archivos con SFTP

Aunque usar Git es un método moderno para desplegar código, es importante conocer métodos tradicionales como SFTP (SSH File Transfer Protocol). SFTP utiliza el protocolo SSH para transferir archivos de forma segura y cifrada.

No necesitas instalar un servidor FTP, ya que SFTP funciona sobre el servidor OpenSSH que ya tenemos instalado. Solo necesitas un cliente gráfico como Filezilla en tu máquina local.

Desplegar un nuevo sitio web mediante SFTP

Configura un nuevo dominio (ej. actividad-24) repitiendo los pasos de la actividad 2.2, pero esta vez transfiere los archivos del nuevo sitio web mediante SFTP.

En Filezilla, para conectar a tu servidor:

  1. En Servidor, introduce sftp://IP_DEL_SERVIDOR.
  2. Introduce tu nombre de usuario.
  3. Como usas autenticación por clave, ve a Edición > Opciones > SFTP y añade tu archivo de clave privada (.pem o id_rsa).

open for write: permission denied

Puede que, al intentar probarlo, te aparezca el error open for write: permission denied. Esto es algo normal si no tienes los permisos de usuario adecuados, ya sea porque seguiste el manual de la actividad 2.2 o porque lo hiciste con el usuario de tu propia máquina.

Lo que debes comprobar son los permisos de la carpeta que has creado y con qué usuario la creaste. Además, verifica también con qué usuario estás intentando conectarte en FileZilla.

Una vez conectado, sube los archivos al directorio /var/www/actividad-24/html y configura el nuevo Server Block.
