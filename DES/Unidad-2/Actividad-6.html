Actividad 2.6: Creación de un Servidor VPS en AWS

Para crear una máquina virtual (servidor VPS) con Debian en Amazon Web Services (AWS), utilizarás el servicio Amazon EC2 (Elastic Compute Cloud).

Aquí tienes la guía paso a paso para lanzar una instancia EC2 con Debian:

Paso 1: Configuración de la Cuenta y Acceso a EC2

  • Regístrate en AWS: Si no lo has hecho, crea una cuenta de AWS. Necesitarás una tarjeta de crédito, aunque si usas el Free Tier (nivel gratuito), no se te cobrará nada por la instancia básica durante 6 meses o los 100$ de crédito.
  • Accede a la Consola EC2:
    1. Inicia sesión en la Consola de AWS.
    2. Busca "EC2" en la barra de búsqueda superior y haz clic en el servicio.

Paso 2: Iniciar la Instancia de Debian

En el panel de EC2, haz clic en el botón naranja "Launch Instance" (Lanzar instancia).

Paso 3: Configuración de la Instancia

Rellena los siguientes pasos en el asistente de lanzamiento:

  1. Nombre y Etiqueta (Name and Tags)

    Name: Asígnale un nombre fácil de recordar, por ejemplo: Servidor-Debian-Proyectos.

  2. Imagen de la Aplicación y del Sistema Operativo (Application and OS Images - AMI)

    Aquí es donde seleccionas Debian.

    En la sección "Quick Start", busca y selecciona una AMI que diga "Debian". Asegúrate de que sea la versión que deseas (ej: Debian 11 o 12) y que esté marcada como "Free tier eligible" (elegible para el nivel gratuito).

  3. Tipo de Instancia (Instance Type)

    Selecciona t2.micro o t3.micro. Estos son los tipos de instancia que califican para el Nivel Gratuito de AWS. Son suficientes para empezar.

  4. Par de Claves (Key Pair - Login)

    Esta es la parte más importante para el acceso de seguridad. AWS utiliza pares de claves SSH para que puedas acceder a tu servidor, en lugar de contraseñas.

    Selecciona una existente o crea una nueva: Si no tienes una, selecciona "Create a new key pair".

    • Nombre: Asígnale un nombre (ej: mi-clave-debian).
    • Tipo: Deja RSA.
    • Formato de Archivo Privado: Selecciona .pem (para usuarios de Linux/macOS) o .ppk (si planeas usar PuTTY en Windows).

    Haz clic en "Create key pair". Guarda este archivo .pem en un lugar seguro; lo necesitarás para conectarte.

  5. Configuración de Red (Network Settings)

    Firewall (Security Groups): Esta es la configuración del Firewall de AWS.

    Haz clic en "Create security group" (Crear grupo de seguridad).

    • Regla 1 (SSH): Deja la regla por defecto para el puerto 22 (SSH). En la columna "Source type" (Tipo de origen), selecciona "My IP" (Mi IP) para mayor seguridad, o "Anywhere" (Cualquier lugar - 0.0.0.0/0) si tu IP cambia a menudo.
    • Regla 2 (HTTP/Web): Haz clic en "Add rule" (Agregar regla).
      • Type (Tipo): HTTP
      • Source (Origen): Anywhere (Cualquier lugar - 0.0.0.0/0)
    • Regla 3 (Nginx/Puerto 8080): Si usas el puerto 8080 para Nginx, agrega otra regla:
      • Type (Tipo): Custom TCP
      • Port range (Rango de puertos): 8080
      • Source (Origen): Anywhere (Cualquier lugar - 0.0.0.0/0)
  6. Configurar Almacenamiento (Configure Storage)

    Deja el tamaño predeterminado (generalmente 8 GB), que es suficiente para el nivel gratuito.

  7. Lanzar Instancia

    Revisa el resumen y haz clic en el botón naranja "Launch instance" (Lanzar instancia).

Paso 4: Conectarse a la Instancia

Una vez que la instancia muestre el estado "Running" (En ejecución), selecciónala en el panel de EC2.

Haz clic en el botón "Connect" (Conectar).

AWS te mostrará el comando SSH para conectarte. Será similar a:

ssh -i /ruta/a/mi-clave-debian.pem admin@<IP-Pública-de-tu-Instancia>

(Nota: El usuario predeterminado para Debian en AWS es a menudo admin o ec2-user, AWS te lo indicará).

Ejecuta este comando en tu terminal (después de cambiar los permisos del archivo de clave):

chmod 400 /ruta/a/mi-clave-debian.pem
ssh -i /ruta/a/mi-clave-debian.pem admin@<IP-Pública-de-tu-Instancia>

¡Ya estarás dentro de tu servidor Debian en AWS!
