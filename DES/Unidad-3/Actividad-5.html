Introducción

En esta actividad vamos a trabajar con Django, un framework web de alto nivel para Python que fomenta el desarrollo rápido y el diseño limpio y pragmático. Django sigue el principio "baterías incluidas", lo que significa que viene con muchas herramientas listas para usar.

¿Qué es Django?

Django es un framework web gratuito y de código abierto escrito en Python. Sigue el patrón de arquitectura Modelo-Vista-Template (MVT). Es utilizado por sitios como Instagram, Pinterest y Mozilla.

Diferencias con Flask

Mientras que Flask es un micro-framework que te da lo mínimo y tú eliges las herramientas, Django es un framework "full-stack" que incluye ORM, autenticación, panel de administración y más, todo integrado.

1. Instalación y Configuración

1.1 Preparar el entorno

Asumimos que ya tienes Python y pip instalados de la actividad anterior. Si no, repasa la Actividad 3.2.

  1. Crea un directorio para tu proyecto Django: 
    mkdir ~/mi-proyecto-django
cd ~/mi-proyecto-django
  2. Crea y activa un entorno virtual: 
    python3 -m venv venv
source venv/bin/activate
  3. Instala Django y Gunicorn: 
    pip install django gunicorn
  4. Verifica la instalación: 
    python -m django --version

2. Creación del Proyecto

2.1 Iniciar un nuevo proyecto

  1. Genera la estructura del proyecto: 
    django-admin startproject misitio .
    Nota: El punto . al final es importante para crear el proyecto en el directorio actual y no crear una subcarpeta extra.
  2. Aplica las migraciones iniciales (base de datos): 
    python manage.py migrate
  3. Crea un superusuario (para el panel de administración): 
    python manage.py createsuperuser
    Sigue las instrucciones para establecer usuario, email y contraseña.
  4. Permite el acceso desde cualquier host (para desarrollo): 
    nano misitio/settings.py
    Busca ALLOWED_HOSTS y cámbialo a: 
    ALLOWED_HOSTS = ['*']
  5. Prueba el servidor de desarrollo: 
    python manage.py runserver 0.0.0.0:8000
  6. Accede desde tu navegador a:
    • http://IP-DE-TU-SERVIDOR:8000 (Verás el cohete de Django)
    • http://IP-DE-TU-SERVIDOR:8000/admin (Panel de administración)

3. Crear una Aplicación "Hola Mundo"

En Django, un "proyecto" puede contener varias "aplicaciones". Vamos a crear una simple.

  1. Detén el servidor (Ctrl+C).
  2. Crea la app: 
    python manage.py startapp hola
  3. Define una vista en hola/views.py: 
    from django.http import HttpResponse

def index(request):
    return HttpResponse("¡Hola Mundo desde Django en Debian!")
  4. Crea el archivo hola/urls.py: 
    from django.urls import path
from . import views

urlpatterns = [
    path('', views.index, name='index'),
]
  5. Configura la URL en misitio/urls.py: 
    from django.contrib import admin
from django.urls import path, include

urlpatterns = [
    path('admin/', admin.site.urls),
    path('', include('hola.urls')),
]
  6. Modificar el archivo misitio/settings.py, añadir la linea hola en INSTALLED_APPS. 
    INSTALLED_APPS = [
    'django.contrib.admin',
    'django.contrib.auth',
    'django.contrib.contenttypes',
    'django.contrib.sessions',
    'django.contrib.messages',
    'django.contrib.staticfiles',
    'hola',
]
  7. Vuelve a ejecutar el servidor y verifica que en la raíz ahora sale tu mensaje.

4. Despliegue con Gunicorn y Nginx

4.1 Archivos Estáticos

Django no sirve archivos estáticos (CSS, JS, imágenes) eficientemente en producción. Necesitamos recolectarlos para que Nginx los sirva.

  1. Edita misitio/settings.py y añade al final: 
    import os
STATIC_ROOT = os.path.join(BASE_DIR, 'static')
  2. Ejecuta el comando para recolectar estáticos: 
    python manage.py collectstatic
    Confirma con 'yes'.

4.2 Configurar Systemd y Gunicorn

  1. Prueba Gunicorn: 
    gunicorn --bind unix:/home/TuUsuario/mi-proyecto-django/misitio.sock misitio.wsgi:application
    Nota: Usamos un socket Unix en lugar de un puerto TCP.
  2. Crea el servicio systemd: 
    sudo nano /etc/systemd/system/django-app.service
    [Unit]
Description=Gunicorn daemon for Django Project
After=network.target

[Service]
User=TuUsuario
Group=www-data
WorkingDirectory=/home/TuUsuario/mi-proyecto-django
ExecStart=/home/TuUsuario/mi-proyecto-django/venv/bin/gunicorn --access-logfile - --workers 3 --bind unix:/home/TuUsuario/mi-proyecto-django/misitio.sock --umask 007 misitio.wsgi:application

[Install]
WantedBy=multi-user.target
    Permiso de Socket: El --umask 007 permite que el socket sea accesible por el grupo www-data, que es necesario para Nginx.
  3. Inicia y habilita el servicio: 
    sudo systemctl daemon-reload
sudo systemctl start django-app
sudo systemctl enable django-app

4.3 Configurar Nginx

  1. Crea la configuración de Nginx: 
    sudo nano /etc/nginx/sites-available/django-app
    server {
    listen 80;
    server_name tu-dominio.com;

    location = /favicon.ico { access_log off; log_not_found off; }

    location /static/ {
        root /home/TuUsuario/mi-proyecto-django;
    }

    location / {
        include proxy_params;
        proxy_pass http://unix:/home/TuUsuario/mi-proyecto-django/misitio.sock;
    }
}
  2. Activa el sitio y reinicia Nginx: 
    sudo ln -s /etc/nginx/sites-available/django-app /etc/nginx/sites-enabled/
sudo systemctl restart nginx
  3. Asegúrate de que Nginx pueda acceder al socket: 
    sudo chmod 750 /home/TuUsuario
sudo chown -R TuUsuario:www-data /home/TuUsuario/mi-proyecto-django
sudo chmod 660 /home/TuUsuario/mi-proyecto-django/misitio.sock
    Nota: Los permisos correctos en el directorio padre y el socket son necesarios para evitar el error 502 Bad Gateway.
