Configuración Inicial del Servidor

1. Configurar el hostname (nombre del servidor)

El siguiente comando permite establecer el nombre de la máquina:

hostnamectl set-hostname servidor-despliegue

¿Qué hace?

Cambia el nombre interno del sistema (hostname), que es como el equipo se identifica a sí mismo y a otros dentro de una red.

Por ejemplo, si configuras el nombre a servidor-despliegue, cuando abras la terminal verás un prompt similar a este:

despliegue@servidor-despliegue:~$

Además, si ejecutas ping servidor-despliegue, el sistema sabrá que se refiere a tu propia máquina local.

Consejos

  • Usa nombres descriptivos y simples, sin espacios ni caracteres especiales.
  • Ejemplos válidos: srvweb, debian01, vps-pruebas, despliegue-daw.

2. Configurar la zona horaria

Para ajustar la hora del sistema a tu región (en este caso, España peninsular), utiliza el siguiente comando: 

timedatectl set-timezone Europe/Madrid

¿Qué hace?

Ajusta la zona horaria del sistema operativo. Esto es muy importante por varias razones:

  • Los registros del sistema (logs) tendrán la fecha y hora correctas, facilitando la depuración de problemas.
  • Las tareas programadas (cronjobs) se ejecutarán en el momento esperado.
  • Las aplicaciones web, bases de datos y otros servicios mostrarán fechas y horas coherentes con tu horario local.

Puedes comprobar la configuración de fecha y hora actual del sistema con:

timedatectl

El resultado será similar a este:

               Local time: mié 2025-10-15 09:37:54 CEST
           Universal time: mié 2025-10-15 07:37:54 UTC
                 Time zone: Europe/Madrid (CEST, +0200)
 System clock synchronized: yes
