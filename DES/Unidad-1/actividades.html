Unidad 1. Simulación de Entornos Remotos con VPS y SSH

Actividad - Realizar un manual de instalación de debian

Ahora os mostraré una guía de como hay que instalar Debian 13, con este ejemplo vosotros tenéis que hacer lo mismo, pero en vuestro equipo.

Iniciamos buscando la imagen ISO de Debian 13 (64-bit) en Google, asegurándonos de seleccionar el enlace oficial de descarga de Debian.org.

Abrimos Oracle VM VirtualBox y nos encontramos con la interfaz principal del administrador. Aquí es donde crearemos y gestionaremos nuestras máquinas virtuales.

Empezamos la creación de la nueva máquina virtual. Le asignamos un nombre ("Debian13"), elegimos la carpeta de destino y seleccionamos la imagen ISO de Debian 13 descargada. VirtualBox detecta automáticamente el tipo de SO como Debian (64-bit).

Configuramos la instalación desatendida. Establecemos el nombre de usuario ("antonio") y una contraseña segura ("12345678"). También definimos el nombre de la máquina ("Debian13") y el dominio.

Ajustamos los recursos de hardware para nuestra máquina virtual. En este caso, asignamos 4096 MB de memoria base (RAM) y 4 CPUs virtuales para un rendimiento óptimo.

Creamos un nuevo disco duro virtual. Establecemos un tamaño de disco de 20.00 GB, suficiente para la instalación y futuras aplicaciones de Debian.

Revisamos el resumen de todas las configuraciones realizadas. Confirmamos que el nombre, la carpeta, la imagen ISO, los datos de usuario y el hardware sean correctos antes de finalizar la creación de la máquina virtual.

La máquina virtual Debian 13 inicia. Se muestra el terminal de arranque con algunos mensajes del sistema. Observamos que la integración del ratón está activa.

Durante el proceso de instalación, se cargan componentes adicionales necesarios para el sistema operativo, como módulos inalámbricos, mostrando el progreso de la descarga.

Una vez completada la instalación, llegamos a la pantalla de inicio de sesión de Debian 13, donde debemos introducir la contraseña para el usuario "antonio".

Después de iniciar sesión, somos recibidos con una ventana de bienvenida a Debian GNU/Linux 13 (Trixie), ofreciéndonos la opción de saltar el tour o realizarlo.

Accedemos a la configuración del sistema para ajustar el idioma y formato regional. Cambiamos la configuración para que todo el sistema y la pantalla de inicio de sesión utilicen el español.

En la configuración de teclado, verificamos que la fuente de entrada principal sea "Español" y configuramos las preferencias de teclado para el sistema.

En la configuración de red, vamos a Red -> Adaptador 1 y lo dejamos en Conectado a: Adaptador Puente.
Esto evita que tengamos problemas a la hora de conectarnos a nuestro servidor Debian.

Abrimos la terminal de Debian. La línea de comandos muestra el usuario actual ("antonio") y el nombre de la máquina ("Debian13"), listo para ejecutar comandos.

Utilizamos el comando ip a en la terminal para verificar las direcciones IP de la máquina virtual. Aquí se muestra la dirección IP local asignada a la interfaz de red, por ejemplo 192.168.1.28.

Accedemos como superusuario (root) con el comando su root e inmediatamente después ejecutamos /usr/sbin/visudo para editar el archivo sudoers, lo que nos permite configurar privilegios de sudo.

Dentro del editor nano, editamos el archivo sudoers. Añadimos la línea antonio ALL=(ALL:ALL) ALL para otorgar al usuario "antonio" permisos completos de sudo, permitiéndole ejecutar cualquier comando como root.

Desde nuestra máquina anfitriona, intentamos conectarnos a la VM de Debian mediante SSH usando ssh -l antonio 192.168.1.28. El primer intento falla con un mensaje de "Connection refused", indicando que el servidor SSH aún no está instalado o en funcionamiento.

De vuelta en la terminal de Debian, ejecutamos sudo apt update para actualizar la lista de paquetes y luego sudo apt install openssh-server para instalar el servidor OpenSSH, que habilitará las conexiones SSH.

Finalmente, intentamos la conexión SSH de nuevo. Tras aceptar la huella digital del host y proporcionar la contraseña del usuario "antonio", la conexión es exitosa, confirmando el acceso remoto a nuestra máquina virtual Debian 13.
